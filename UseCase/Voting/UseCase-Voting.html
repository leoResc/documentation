1. Use Case Name

Voting

1.1 Brief Description

A logged in user clicks on the like-button and gets a visual response.

2. Flow of Events

2.1 Basic Flow

The logged in user clicks the like-button. If he didn't like the song before, the button gets red and the song gets +1 vote. If he liked the song before, the button gets its original colour and the song gets -1 vote.

    Casual User likes not liked song

  1. User clicks on like-button
  2. Server validates that user hasn't voted for this song yet.
  3. votes +1 will be saved in a database
  4. User gets visual response to his "like action".

2.2 Alternative Flows

2.2.1 Fist Alternative Flow

    Casual User likes already liked song

  1. User clicks on like-button
  2. Server validates that user has already voted for this song.
  3. votes -1 will be saved in a database
  4. User gets visual response to his "like action".

3. Special Requirements

No special requirements are needed

4. Preconditions

The user has to be logged in.

5. Postconditions

5.1 Postcondition One

The user gets visual response for voting.

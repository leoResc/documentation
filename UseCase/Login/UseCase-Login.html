1. Use Case Name

Login

1.1 Brief Description

A user can login with a personal nickname and the password for the current session.

2. Flow of Events

2.1 Basic Flow

The user enters his nickname and the according password. A click on the login button will send the information to the server. There they will be verified.

    Administrator

  1. Username has to be "admin"
  2. Password has to be the right password for administrator account
  3. User will be connected to the admin panel

    Casual User

  1. User chooses a nickname for himself
  2. Server validates that username is not assigned to another user yet.
  3. Username will be safed in a database
  4. Password has to be the right password for the current session
  5. Then the user will be connected to the user page

Link to according .feature file

2.2 Alternative Flows

2.2.1 Fist Alternative Flow - Wrong admin password

If the user is admin and the entered password is incorrect, then the user gets the message "wrong username or password".

2.2.2 Second Alternative Flow - User exists already

If the username exists already, then the user gets the message "user exists already".

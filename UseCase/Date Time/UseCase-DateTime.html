1. Use Case Name

Date Time

1.1 Brief Description

As the Raspberry Pi loses power whenever it is shutdown, it also loses it's current date and time. Because the server is not connected to the internet, it cannot retrieve the current time.
Creating events with a according time frame is therefore absurd. The Raspberry Pi needs to know the urrent time in order to activate the correct event.

2. Flow of Events

2.1 Basic Flow

The admin sees the time stamp of the Raspberry Pi on the dashboard. When he clicks on the button, which indicates a time actualisation, the time on the Raspberry Pi will be updated.
This action is realised by a date object created with javascript on the client side. This date object contains the current time of the client, which is usually correct. This date object is then transmitted to the server. The server then updates it's time.

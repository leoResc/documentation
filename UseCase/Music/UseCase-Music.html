1. Use Case Name

Music

1.1 Brief Description

The admin can let the Raspberry Pi play the music of CLOUD. The admin can also pause the music and forward to the next song.
Moreover, the admin can change the volume of the music.
The logic controlling which song has to be played next is also part of this use case.

2. Flow of Events

2.1 Basic Flow

When the admin pushes the play button, the Raspberry Pi starts playing the music of CLOUD. The admin can also pause the music and change to the next song.
Moreover, the admin can change the volume of the music.The logic controlling which song has to be played next is also part of this use case. The song with the most likes will be added to the current playlist as soon as the last song is about to end.
The admin sees the time stamp of the Raspberry Pi on the dashboard. When he clicks on the button, which indicates a time actualisation, the time on the Raspberry Pi will be updated.
This action is realised by a date object created with javascript on the client side. This date object contains the current time of the client, which is usually correct. This date object is then transmitted to the server. The server then updates it's time.

4. Preconditions

4.1 Precondition One

The user needs to be logged in as admin in order to change the volume of CLOUD or to play/pause/forward music. Moreover, the admin needs to be on the page 'Dashboard'.

5. Postconditions

5.1 Postcondition One

The user can hear that the volume of the music changed or that music stopped / started / paused or that the Raspberry Pi plays the next song.

1. Use Case Name

Audio Visualisation

1.1 Brief Description

Micro controller, Arduino Uno, gets command to react to noise / sound.

2. Flow of Events

2.1 Basic Flow

No Mockup needed, because visual interaction won't be available to admin nor casual user.

The micro controller gets the "HIGH" signal for "react to sound" and starts his algorithm for visualisation with the LEDs.

    Arduino gets "LOW -> HIGH" for "react to sound"

  1. Arduino gets noise level from outer space
  2. Arduino starts algorithm for visualisation
  3. Visualisation programm light up LEDs
  4. Arduino checks for signal from "react to sound"

2.2 Alternative Flows

    Arduino gets "HIGH -> LOW" for "react to sound"

  1. Arduino sets all LEDs to LOW
  2. Arduino sets itself to sleep mode.
  3. Arduino checks for signal from "react to sound"

3. Special Requirements

3.1 Special Requirement 1

3.2 Special Requirement 2

Connection to Raspberry Pi.

3.3 Special Requirement 3

Connection to LEDs.
